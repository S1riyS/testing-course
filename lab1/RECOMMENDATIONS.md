#### 1. Отчёт по лабораторной работе

Отчёт — это важная часть работы, демонстрирующая процесс и результаты тестирования.

**1.1. Методика тестирования**
В начале отчёта необходимо описать **методику тестирования**:

- **Что тестируется:** например, функциональность интернет-магазина.
- **Требования:** на основе каких критериев проводится проверка.
- **Окружение:** условия, в которых выполняется тестирование (ПО, оборудование).
- **Подход:** последовательность действий (например, по сценариям использования — юзкейсам).

**1.2. Структура отчёта**
Обязательные пункты отчёта:

1.  **Методика тестирования** (см. выше).
2.  **Факт выполнения:** документирование самого процесса запуска тестов.
3.  **Результаты тестирования (статистика):**
    - Для **ручных тестов**: список выполненных проверок с указанием шагов.
    - Для **автоматизированных тестов**: отчёт фреймворка (например, `JUnit`).
    - **Анализ неудачных тестов:** необходимо объяснять причины сбоев (например, "сетевые проблемы", "несоответствие ожидаемому результату").

**1.3. Оформление**
Строгие стандарты (ГОСТы) не обязательны. Допускается свободная форма, но отчёт должен быть структурированным, полным и понятным.

**1.4. Порядок сдачи лабораторной работы**
Работы проверяются в следующей последовательности:

1.  Отчёт.
2.  Код/реализация.
3.  Ответы на теоретические вопросы.

#### 2. Теоретическая часть

Для успешной сдачи необходимо владеть следующими темами:

**2.1. Виды тестирования**
Необходимо понимать различия между основными типами, например:

- Функциональное и нефункциональное тестирование.
- Сквозное (End-to-End, E2E) тестирование.
- Вид тестирования, указанный в конкретной лабораторной работе, должен быть изучен глубоко.

**2.2. Инструменты и технологии курса**
Требуется знание основ и принципов работы:

- Фреймворк для модульного тестирования — **`JUnit`**.
- Инструмент для автоматизации веб-приложений — **`Selenium`**.
- Инструмент для нагрузочного тестирования — **`Apache JMeter`**.
